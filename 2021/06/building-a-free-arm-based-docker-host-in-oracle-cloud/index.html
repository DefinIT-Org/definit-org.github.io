BUILDING A FREE ARM-BASED DOCKER HOST IN ORACLE CLOUD

Written by Sam McGeown on 21/6/2021 · Read in about 4 min (847 words)
Published under Cloud-Native and Cloud
Tags: #docker #oracle #free

Most people have used AWS' free tier of resources for learning - and most people (including myself) have long since used the 12 months allowed free tier. It was recently brought to my attention that Oracle Cloud have a free tier offering (that they promise is always free) that includes two AMD-based compute VMs with 1/8th of an CPU and 1GB RAM (not much use to anyone) and (much more interestingly) a 4 Arm-based Ampere A1 cores and 24GB usable RAM as 1-4 VMs. This got me thinking about whether I could create a Docker host for vRealize Automation Code Stream to run my CI tasks.

https://www.oracle.com/uk/cloud/free/

Once you’ve signed up and verified your account, it takes a few minutes to fully provision your cloud account - you’ll get an email when it is complete.

I named my new VM oc-docker-host and left the default placement settings.

I edited the image and shape settings, then clicked “Change Shape” and selected the Ampere processor. From there I could select the VM.Standard.A1.Flex Shape, and expand the settings to allow access to the CPU and memory sliders - needless to say I cranked those up to MAX! Just make sure you’re seeing the “Always Free Eligible” tag.

Once the shape is selected, the image will automatically be Oracle Linux 7.9, which is one of the restrictions for the “Always Free” tier.

I left the networking configuration as is, which by default will assign a public IP to the VM

I uploaded one my public key to use with SSH authentication by uploading a .pub file

And finally, I left the boot volume options at default (46.6GB)

Finally, I hit “Create” to launch my new Arm-based VM. It took 20-40 seconds to boot and become fully available to me, but once it was up I could SSH to the public IP address.

Installing Docker

Oracle Linux maintain a package for docker in their public repositories, so I simply ran a sudo yum update to ensure everything was up to date, then sudo yum install docker-engine, which installed the required packages and docker runtime.

 1sudo yum install docker-engine -y
 2sudo systemctl start docker
 3sudo systemctl enable docker
 4sudo systemctl status docker

 6   Loaded: loaded (/usr/lib/systemd/system/docker.service; enabled; vendor preset: disabled)

 8     Docs: https://docs.docker.com

Then, based on the steps I blogged about (here)[https://blogs.vmware.com/management/2020/08/creating-a-docker-host-for-vra-code-stream.html] I configured the Remote API and TLS.

 1export HOST="oc-docker-host"
 2export IP="132.145.54.183"
 3sudo mkdir /etc/docker/ssl
 4sudo openssl genrsa -aes256 -out ca-key.pem 4096
 5# Enter a password for the CA key
 6sudo openssl req -new -x509 -days 365 -key ca-key.pem -sha256 -out ca.pem
 7# Enter the relevant info for the CA
 8openssl genrsa -out server-key.pem 4096
 9openssl req -subj "/CN=$HOST" -sha256 -new -key server-key.pem -out server.csr
10echo subjectAltName = DNS:$HOST,IP:$IP,IP:127.0.0.1 >> extfile.cnf
11echo extendedKeyUsage = serverAuth >> extfile.cnf
12openssl x509 -req -days 365 -sha256 -in server.csr -CA ca.pem -CAkey ca-key.pem -CAcreateserial -out server-cert.pem -extfile extfile.cnf
13chmod -v 0400 ca-key.pem server-key.pem
14chmod -v 0444 ca.pem server-cert.pem

Next we need to create an override file for the docker service (sudo systemctl edit docker.service) to append the TLS flags and tcp port for remote API:

1[Service]
2ExecStart=
3ExecStart=/usr/bin/dockerd -H fd:// -H tcp://0.0.0.0:2376 --tls --tlscacert=/etc/docker/ssl/ca.pem --tlscert=/etc/docker/ssl/server-cert.pem --tlskey=/etc/docker/ssl/server-key.pem

Restart the docker service sudo systemctl daemon-reload && sudo systemctl restart docker.service

Configure network access

First, we need to configure the host firewall to allow access to Docker

Next, we need to configure the Oracle Cloud network security. Starting from the Instance Details page of my VM, I click on the Subnet to go through to the Virtual Cloud Network subnet details page. From there I can click on the Default Security List and add ingress rules for my Docker host (API and Code Stream Agent).
